brand anchor: `#2a2320` (primary dark, confirmed from theme-color meta).

Group all found values into:
- **Core palette** — appears 3+ times, clearly intentional
- **Accent values** — appears 1–2 times, may be intentional or may be drift
- **One-offs** — appears once, likely unintentional

**6.3 — Implement CSS custom properties**

If a `:root {}` block with CSS custom properties does not already exist at the top of
`/assets/css/main.css` (or the primary global stylesheet), add one. It must include at

Use this structure as the starting point, populated with the actual values found:

```css

   Updated: May 2026
   ===================================================== */
:root {
  /* --- Color Palette --- */

  --color-bg-secondary:     [value];   /* secondary background */
  --color-bg-surface:       [value];   /* card / surface background */
  --color-text-primary:     [value];   /* primary body text */
  --color-text-secondary:   [value];   /* muted / secondary text */
  --color-text-heading:     [value];   /* heading text */
  --color-accent-primary:   [value];   /* primary brand accent */
  --color-accent-secondary: [value];   /* secondary accent */
  --color-border:           [value];   /* border / divider color */
  --color-link:             [value];   /* default link color */
  --color-link-hover:       [value];   /* link hover state */

  /* --- Typography --- */
  --font-heading:  [value];            /* heading font stack */
  --font-body:     [value];            /* body font stack */
  --font-mono:     [value];            /* monospace / code font stack */

  /* --- Spacing Scale --- */
```

**6.4 — Replace hardcoded values**

After the `:root` block is in place, do a find-and-replace pass across all CSS files.
Replace every hardcoded instance of a core palette color with its corresponding custom
property variable.

Scope this carefully:
- Replace only the core palette colors (3+ occurrences) — do not touch one-off values
- Do not replace color values inside vendor-prefixed blocks or third-party code
- After replacing, verify the site renders correctly by checking that no replaced value
  produced a broken or `unset` result

**6.5 — Inline style cleanup**

Scan all HTML files for `style=""` attributes containing color values. For each one:
- If the color is a core palette color: replace the inline style with the appropriate
  CSS class (create the class in main.css if it doesn't exist)
- If the color is a one-off: flag it in the report for human review; do not change it
