## ROOT CAUSE ANALYSIS (confirmed from screenshots and source HTML)

### Failure 1 — Status Board item overflow

**Component:** `.status-board` / `.status-item`
**What’s happening:** Status items that contain inline `<code>` elements with long
path strings (e.g. `/Volumes/OKH-Local/07_Local_LLMs/ollama/models`) are overflowing
their container because `code` elements do not wrap by default. The path text is
rendering outside the card boundary and getting clipped.

**Affected items (confirmed from HTML source):**

- `<code>q8_0</code>` KV cache line
- `<code>/Volumes/OKH-Local/07_Local_LLMs/ollama/models</code>`
- `<code>/Volumes/OKH-Local/07_Local_LLMs/huggingface-cache</code>`
- `<code>.zprofile</code>` lines (multiple)
- `<code>localhost:3000</code>` line

### Failure 2 — Key Decisions sidebar card overflow

**Component:** `.decision-item` (inside the sidebar Key Decisions block)
**What’s happening:** Each `.decision-item` appears to use a flex row layout with
`.decision-key` and `.decision-val` as siblings. On mobile, the value text is
overflowing or being clipped at the right edge of its container. The key/value
pairs need to stack vertically on mobile.

**Affected items (confirmed from screenshots):**

- MODEL POLICY
- TOKEN GOVERNANCE
- GIT WORKFLOW
- DIRECTORY CONVENTIONS
- (and all other `.decision-item` rows)

### Failure 3 — Data tables (inferred from HTML, confirmed as mobile risk)

**Component:** `.data-table` inside `.data-table-wrap`
**What’s happening:** The Model Inventory and Benchmark tables contain columns
that will overflow at narrow viewports if the wrapper does not have
`overflow-x: auto`. This may or may not be currently clipping — fix regardless
as it is a confirmed mobile table pattern failure.

### Failure 4 — Timeline rows (inferred from HTML source)

**Component:** `.timeline-row` with `.timeline-row-label` (Did/Worked/Broke badges)
and `.timeline-row-content`
**What’s happening:** If `.timeline-row` is a flex row, the label badge and content
text need to wrap properly at narrow widths. Long content strings inside
`.timeline-row-content` may be overflowing.

### Failure 5 — Banner label stacking (visible in Screenshot 1)

**Component:** The hot-forge announcement banner
**What’s happening:** At ~375px the banner label text (“HOT OFF THE FORGE 🔥”)
is stacking vertically and the link text is wrapping, creating a tall banner block
that takes significant vertical space. This is less critical than the overflow
issues but should be tightened.

## TASK 2: FIX FAILURE 1 — CODE ELEMENT WORD WRAPPING

Add the following CSS rules. If a rule for the selector already exists, add the
missing properties to the existing rule rather than creating a duplicate.

```css
/* Mac Studio page — code element overflow fix */
/* Allows long path strings to wrap inside status items and decision cards */

code {
  word-break: break-word;
  overflow-wrap: break-word;
  white-space: pre-wrap; /* preserves spacing but allows wrapping */
}

.status-item code,
.decision-item code,
.callout code,
.timeline-row-content code {
  word-break: break-all;   /* aggressive break for long paths with no natural break points */
  overflow-wrap: anywhere;
  white-space: normal;
  display: inline;
}
```

**Why `break-all` for these specific contexts:** The paths contain no natural
word-break characters (no spaces, no hyphens in the right places). `break-word`
alone will not break them. `break-all` forces a break at the container boundary.

## TASK 3: FIX FAILURE 2 — DECISION ITEM MOBILE STACKING

The `.decision-item` flex row must collapse to a vertical stack at mobile widths.

First, read the current `.decision-item` CSS to understand its current flex
direction and gap. Then apply:

```css
/* Mac Studio page — decision item mobile stacking */
@media (max-width: 640px) {
  .decision-item {
    flex-direction: column;
    gap: 0.25rem;
    align-items: flex-start;
  }

  .decision-key {
    width: 100%;
    max-width: 100%;
    flex-shrink: 0;
  }

  .decision-val {
    width: 100%;
    word-break: break-word;
    overflow-wrap: break-word;
    padding-left: 0; /* remove any left indent that assumes key is beside it */
  }
}
```

If `.decision-key` or `.decision-val` have explicit `width` or `min-width` values
that would prevent proper stacking, override them inside the same media query.

## TASK 4: FIX FAILURE 3 — TABLE OVERFLOW

Verify that `.data-table-wrap` has the following CSS. If it does, confirm it.
If it does not, add it:

```css
.data-table-wrap {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
  width: 100%;
  max-width: 100%;
}

.data-table {
  min-width: 500px; /* prevents table collapse below readable column widths */
  width: 100%;
  border-collapse: collapse;
}
```

The `min-width: 500px` on `.data-table` paired with `overflow-x: auto` on the
wrapper is the correct pattern — the table scrolls horizontally rather than
collapsing into an unreadable stack.

Additionally, add to the Model Inventory table cells:

```css
.data-table td code {
  word-break: break-all;
  overflow-wrap: anywhere;
  white-space: normal;
}
```

This prevents model name code strings from overflowing table cells on the
horizontal scroll layer.
