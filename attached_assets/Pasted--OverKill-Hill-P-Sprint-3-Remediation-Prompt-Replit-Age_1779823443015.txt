## TASK 8: RESPONSIVE DESIGN TESTING AND FIXES

**Priority: High | Risk: Low**

Use Playwright, browser preview, or device emulation at these widths:
320px, 375px, 428px, 768px, 1024px, 1440px.

**8.1 — Hamburger button text visibility**

At 375px on every page: confirm "Toggle navigation" is NOT rendering as visible
on-screen text. It must be hidden visually but present for screen readers.

If raw text is rendering on screen, replace every hamburger button with:
```html
<button class="nav-toggle" aria-label="Toggle navigation" aria-expanded="false">
  <span class="visually-hidden">Toggle navigation</span>
  <span aria-hidden="true">&#9776;</span>
</button>
```

Ensure this CSS class exists in the global stylesheet:
```css
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}
```

**8.2 — BPMN iframe responsiveness at 375px**

On `/projects/bpmn-for-mermaid/`: does the embedded playground iframe overflow
or collapse at 375px?

If the iframe container lacks responsive CSS, add to the page-specific or
global stylesheet:
```css
.bpmn-playground-container,
.playground-embed-wrapper {
  position: relative;
  width: 100%;
  padding-top: 56.25%;
  overflow: hidden;
}
.bpmn-playground-container iframe,
.playground-embed-wrapper iframe {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  border: 0;
}
```
Use the actual class name on the iframe container found in the HTML.

**8.3 — Code block overflow**

On the BPMN page, does the bpmn-beta code example block cause horizontal scroll
at 375px?

If `overflow-x: auto` is not already on `pre` elements globally:
```css
pre {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
  white-space: pre;
}
```

**8.4 — Horizontal scroll check — all pages**

At 375px, verify no page produces horizontal scroll. If any do, add to
global CSS:
```css
body {
  overflow-x: hidden;
}
```
Flag any page where horizontal scroll persists after this fix — it indicates
a fixed-width element that needs targeted correction.

**8.5 — Touch target size**

At 375px, verify that all nav links, buttons, and CTAs have a minimum tap
target height of 44px. If any fall below this, add `min-height: 44px` to
the relevant component class in the global stylesheet.
