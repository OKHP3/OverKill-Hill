SECTION 6: FEATURES GRID

Feature grid must contain at least these cards.

1. Live Diagram Preview
   Body:
   Every color change reflects instantly across representative Mermaid diagram families. The point is not decoration. The point is fast visual governance.

2. Diagram Family Detection
   Body:
   Automatically identifies the diagram family and applies family-aware styling constraints. Not every Mermaid diagram family supports the same styling surfaces.

3. Multi-Format Export
   Body:
   Five export formats: complete themeVariables block, Markdown style bootstrap, renderer-aware prompt scaffold, diff view output, and raw CSS variable block. Each format maps to a different workflow: paste into LLMs, drop into mermaid.initialize(), or commit to a repo.

4. Prompt Scaffold Generator
   Body:
   Generates LLM-ready prompt prefixes with palette, renderer, and diagram-family constraints embedded before generation begins.

5. Renderer Capability Notes
   Body:
   Flags where styling choices break or degrade across GitHub, GitLab, Obsidian, Notion, Confluence, Mermaid Live, and CLI-style renderer contexts.

6. Brand Palette Presets
   Body:
   Presets include Ocean Depth, Forest Sage, Slate Ember, Violet Mist, OverKill Hill P³, Glee-fully, AskJamie, and Neutral Enterprise. Utility palettes are for general use. Brand palettes are governed conventions.

7. Precise Color Controls
   Body:
   Color controls expose exact values and visual feedback. The goal is governed iteration, not prompt-by-prompt guessing.

8. Browser-Only
   Body:
   Runs client-side in the browser. No login, no backend, no analytics, no server-side theme storage.

9. Pan / Zoom Canvas
   Body:
   Navigate dense diagrams without losing context. Useful for complex flowcharts, class diagrams, and multi-section diagrams that do not fit the initial viewport.

10. Forge UI System
   Body:
   The interface runs on OKH Forge design tokens aligned to overkillhill.com. The workbench itself demonstrates the same visual governance it produces.

11. Dark / Light / System Mode
   Body:
   Full dark, light, and system-preference modes with persistent preference. Preview context should reflect the selected mode.

12. Mermaid Look API
   Body:
   Classic, Neo, and Hand-Drawn looks, with renderer support warnings where applicable. The page must not imply every renderer supports every look.

13. Diff View
   Body:
   Side-by-side or equivalent comparison of original and themed Mermaid code so users can audit exactly what changed.

14. SKILL.md Agent Skill
   Body:
   Governance logic packaged for AI coding assistants and agent workflows. The visual app is the workbench. SKILL.md is the headless enforcement layer.

15. Open Source, MIT License
   Body:
   Source available on GitHub. Free to inspect, fork, adapt, and improve under the license terms.

SECTION 12: FAQ CORRECTIONS

Fix or add these FAQ entries.

Q: Is this affiliated with Mermaid.ai?
A:
No. This is an independent community contribution built around open Mermaid.js theming concepts. It is not affiliated with Mermaid.ai, Mermaid Chart, or any commercial Mermaid product.

Q: Is this official Mermaid?
A:
No. It is an independent OverKill Hill P³ community tool. It exports Mermaid-compatible theme configuration, but it is not an official Mermaid project.

Q: Does it cost anything?
A:
No. It is free and MIT licensed.

Q: Does it collect data?
A:
No. The app is client-side. No login, no backend, no analytics, and no server-side theme storage.

Q: Which Mermaid.js version does it target?
A:
Target: Mermaid.js 11.15.0. Exports rely on Mermaid-native themeVariables and are generally portable across Mermaid v10/v11-compatible renderers, but renderer support varies by platform.

Q: Can I use the exported theme with GitHub, Notion, Obsidian, Docusaurus, or VitePress?
A:
Often, but with renderer-specific constraints. The point of the tool is to flag those constraints before you paste the output.

Q: Does it support dark, light, and system mode?
A:
Yes. Dark, light, and system mode shipped before v0.5.0. The preview context should respect the selected mode.

Q: Can I save and share my theme?
A:
Yes. Shareable URL state encoding shipped in v0.5.0. Bookmarking or sharing the URL can preserve the working configuration. Permanent governance artifacts should still be exported and committed to a repo.

Q: Why not just ask an AI for colors?
A:
Because an AI can generate plausible styling without renderer context, contrast discipline, or repeatable governance. This tool gives the model a contract before generation starts.

Q: How does this relate to BPMN for Mermaid?
A:
BPMN for Mermaid is the structural sibling. It explores a text-first bpmn-beta diagram type. Mermaid Theme Builder is the styling and governance layer.

Q: What is SKILL.md here?
A:
SKILL.md packages the same governance rules for agent workflows. The app is interactive. SKILL.md is headless.
