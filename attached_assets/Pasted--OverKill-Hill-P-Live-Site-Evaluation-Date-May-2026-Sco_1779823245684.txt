## RESPONSIVE DESIGN ASSESSMENT

*Note: Full browser-based responsive testing requires Replit-side tooling with Playwright or similar. The following is assessed from markup patterns and known static-site behaviors.*

### What the markup suggests is working:
- `<meta name="viewport" content="width=device-width, initial-scale=1">` is present on all pages
- The hamburger/toggle nav pattern is present on all pages (`Toggle navigation` text visible in rendered output)
- Images use absolute URLs (no path-relative breakage at different directory depths)

### Responsive concerns requiring Replit-side testing:

**R-01 — "Toggle navigation" text visibility**
The hamburger button text "Toggle navigation" appears in the scraped markdown output on every page. If this is rendering as visible text rather than a visually-hidden accessible label, mobile visitors see raw button text. Replit must verify whether this is wrapped in a `visually-hidden` CSS class or `aria-label` and is not rendering as on-screen text at any breakpoint.

**R-02 — BPMN playground iframe at mobile widths**
The BPMN for Mermaid page embeds an iframe playground (`okhp3.github.io/mermaid-diagram-bpmn/`). Iframes require explicit responsive handling — either `width: 100%` with a `padding-top` aspect ratio container, or CSS `aspect-ratio`. At 375px viewport the default iframe behavior will either overflow or render too small to interact with. Replit must test this specifically.

**R-03 — Code block overflow**
The BPMN page includes a multi-line code example block. Code blocks on mobile commonly cause horizontal scroll if `overflow-x: auto` is not applied to the `<pre>` element or its container. Must test at 375px.

**R-04 — Mac Studio status board layout**
The Mac Studio page has a dense checklist with long lines (`✓ Hardware unboxed — KVM, Satechi stand, SN7100 NVMe, Logitech Brio MX`). At 375px these lines will wrap in potentially awkward ways if not governed by appropriate line-height and padding. Verify.

**R-05 — Hero image scaling on homepage**
The 1024×1024 Sentinel image on the homepage should scale down proportionally at mobile widths. Confirm `max-width: 100%` is applied and the image is not forcing a fixed-width layout.
